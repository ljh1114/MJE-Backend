# 데이트 코스 추천 로직 상세 스펙

> BACKLOG.md 1번(추천)과 2번(상세)의 하위 규칙이다.

---

## 1. 시간대 정의

시간대는 5개 구간으로 정의된다.

| 슬롯 | 시간 범위 |
|------|----------|
| `morning` | 09:00 ~ 11:30 |
| `lunch` | 11:30 ~ 14:00 |
| `afternoon` | 14:00 ~ 17:30 |
| `evening` | 17:30 ~ 21:30 |
| `late_night` | 21:30 ~ 01:00 |

---

## 2. 카테고리별 시간대 허용 규칙

카테고리별로 허용되는 시간대가 다르며, 부적합 장소는 추천 후보에서 제외된다.

### 음식점
- `morning` 제외
- `lunch` ~ `evening` 허용
- `late_night`: 술집·포차·야식·심야 키워드를 포함한 장소만 허용

### 카페
- `morning` ~ `evening` 허용
- `late_night` 제외 (심야카페·24시카페 키워드 장소는 예외 허용)

### 산책코스
- `morning` ~ `evening` 허용
- `late_night` 제외

### 이색데이트
- `afternoon` · `evening` 허용
- `late_night`: 실내 장소(영화관·볼링장·노래방)만 허용

---

## 3. 영업시간 필터링 규칙

- 장소의 영업 종료 시각이 해당 시간대 시작 시각 기준으로 **1시간 이내**이면 추천 후보에서 제외된다.
  - 예: `late_night`(21:30 시작) 추천 시, 22:00에 영업 종료하는 식당은 제외
- 영업시간 정보가 제공되지 않는 장소는 제외하지 않는다 (정보 없음으로 처리).

---

## 4. 이미지 필터링 규칙

### 카테고리별 허용 이미지

| 카테고리 | 허용 이미지 유형 |
|---------|----------------|
| 음식점 | 음식 사진, 식당 내부, 식당 외관 |
| 카페 | 음료, 디저트, 카페 내부, 카페 외관 |
| 산책코스 | 실제 장소 풍경, 산책로, 공원, 거리, 야경 |
| 이색데이트 | 체험 공간, 활동 장면, 전시 공간, 실내 데이트 공간 |

### 제외 이미지

아래 조건에 해당하는 이미지는 사용하지 않는다.
- 사람 얼굴이 중심인 이미지
- 광고·협찬 이미지 (`협찬`, `광고`, `제공받아` 키워드 포함)
- 로고, 지도 캡처
- 부동산 이미지 (`부동산`, `분양` 키워드 포함)
- 카테고리와 맞지 않는 이미지

### 유효 이미지 없음 처리

유효 이미지가 존재하지 않으면 `imageUrl` 없이 장소를 포함한다. 장소 자체는 제거하지 않는다.

### 카테고리별 이미지 검색어

| 카테고리 | 검색어 패턴 |
|---------|-----------|
| 음식점 | `{지역명} {장소명} 음식 사진` / `식당 내부` / `맛집 후기` |
| 카페 | `{지역명} {장소명} 카페 외관` / `카페 내부` / `음료 디저트` |
| 산책코스 | `{지역명} {장소명} 산책로` / `공원 풍경` / `야경` |
| 이색데이트 | `{지역명} {장소명} 체험` / `전시` / `실내데이트` |

---

## 5. 추천 코스 구조 규칙

- 추천 코스는 **메인 장소 1곳 + 서브 장소 2곳**으로 구성된다.
- 코스 내 동일 장소(장소명 기준)는 중복 포함되지 않는다.
- 코스 내 동일 카테고리는 중복 포함되지 않는다.

---

## 6. 추천 조합 순서 규칙

코스 순서는 사용자가 선택한 시각을 기준으로 동적으로 결정된다.

### 식사 진입점 기준

식사 진입점은 두 개로 정의된다.

- 점심 진입점: 11:30
- 저녁 진입점: 17:30

### 시작 카테고리 결정 규칙

사용자 선택 시각과 가장 가까운 식사 진입점까지의 거리를 기준으로 첫 번째 장소 카테고리를 결정한다.

| 조건 | 시작 카테고리 |
|------|------------|
| 가장 가까운 식사 진입점까지 **1시간 이내** | 식사 |
| 가장 가까운 식사 진입점까지 **1시간 초과** | 카페 또는 활동 |
| `late_night` (21:30~) | 야식 가능 식당이 있으면 식사, 없으면 활동 |

> 카페와 활동 중 선택은 후보 장소 수가 더 많은 카테고리를 우선한다.

### 카테고리 전환 규칙

시작 카테고리가 결정된 후 나머지 순서는 아래 전환 방향에 따라 구성된다.

| 현재 카테고리 | 다음 카테고리 |
|------------|------------|
| 식사 | 카페 → 활동 |
| 카페 | 활동 → 식사 (식사 진입점이 코스 시간 내에 있을 경우) / 활동 (없으면) |
| 활동 | 식사 진입점 임박 시 식사 → 카페 / 아니면 카페 → 식사 |

### late_night 특수 규칙

- 카페는 심야카페·24시카페 키워드 장소만 포함 가능하며, 없으면 코스에서 제외한다.
- 코스는 식사 → 활동 또는 활동 → 식사 2가지 구조만 허용한다.

### 분위기 일관성 규칙

코스 내 장소 간 분위기 키워드가 일관되어야 한다. 조합 시 키워드 겹침이 가장 많은 순서를 우선 선택한다.

## 7. 응답 구조

### 장소 항목 필드

| 필드 | 타입 | 설명 |
|-----|------|------|
| `type` | `restaurant \| cafe \| walk \| activity` | 카테고리 |
| `name` | string | 장소명 |
| `area` | string | 지역명 |
| `description` | string | 1~2문장 설명 |
| `keywords` | string[] | 분위기·대표 특징·추천 이유 |
| `imageUrl` | string (Optional) | 검증된 이미지 URL, 유효 이미지 없으면 미포함 |
| `recommendedTimeSlot` | string | 추천 시간대 |

### 추천 후보 부족 시 응답

조건에 맞는 장소가 부족하면 아래 형태로 응답한다.

```json
{
  "message": "선택한 조건에 맞는 데이트 코스가 부족합니다. 시간대나 지역을 변경하면 더 많은 추천을 받을 수 있습니다.",
  "recommendations": []
}
```

## 8. 처리 흐름

추천은 아래 순서로 처리된다.

1. 사용자 입력값 분석 (지역, 시간대)
2. 장소 후보 수집 (네이버 검색 API)
3. 시간대 적합성 검사 → 부적합 장소 제거 / 영업 종료 1시간 이내 장소 제거
4. 이미지 후보 검색 및 필터링
5. 데이터랩 트렌드 점수 반영 (정렬 기준으로만 사용, 필터 조건 아님)
6. 추천 조합 생성
